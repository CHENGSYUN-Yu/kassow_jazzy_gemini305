# 自動夾取流程 — 物件使用順序與說明

更新：2026-06-06

---

## 流程範圍

從 YOLO 偵測到夾爪移動至器械上方，共四個階段。

---

## 階段一：座標轉換鏈

觸發方式：`_try_inject_detection()`，由 `tick()` 每 0.3s 呼叫一次。

| # | 物件 | 方法 | 輸入 | 輸出 |
|---|------|------|------|------|
| 1 | **YoloEngine** | `get_dets(0)` | cam0 畫面（GPU 推論） | bbox、center、conf、angle_deg |
| 2 | **DepthReader** | `get_depth(center, depth_frame)` | 像素座標 + 深度圖 | `depth_mm`（5×5 patch 中位數） |
| 3 | **Pixel2HeadCam** | `project_all(dets)` | (u,v) + depth_mm + 相機內參 | `pos_cam_mm: [x,y,z]`（相機座標系） |
| 4 | **HeadCam2Base** | `transform_all(dets)` | pos_cam_mm + T_matrix | `pos_base_mm: [x,y,z]`（base frame） |
| 5 | **Angle2Rz** | `convert_all(dets)` | angle_deg + T_matrix | `yaw_deg`、`Rz 旋轉矩陣` |
| 6 | 內嵌邏輯 | yaw offset + 180° 選優解 | yaw_deg + 當前 TCP yaw | 調整後 yaw_deg（預設 -45° offset） |

### 相機與校正矩陣

| 項目 | 說明 |
|------|------|
| 頭部相機 | D435I（Cam 1，index 0） |
| T_matrix | `T_matrix_20260603_head30.npy`（頭部相機→base frame，4×4） |
| 相機內參 | 連線後從 RealSense SDK 讀取（fx, fy, cx, cy） |
| 深度單位 | uint16，1 unit = 1 mm |

---

## 階段二：選最優先目標

觸發方式：`_try_inject_detection()` 末段，座標轉換完成後。

| # | 物件 | 方法 | 輸入 | 輸出 |
|---|------|------|------|------|
| 7 | **WhichFirst** | `get_first(dets)` | N 個器械（含 pos_base_mm） | 1 個最優先器械 |

**優先順序規則：**
- 主要：`pos_base_mm[2]`（Z）越大越優先（Z 高 = 位置上層 = 先夾）
- 次要：Z 值完全相同時，`pos_base_mm[1]`（Y）越大越優先

排序鍵：`(-z, -y)`，Z 有任何差距都以 Z 決定，Y 僅作為 Z 完全相同時的 tiebreaker。

---

## 階段三：計算接近點

觸發方式：使用者按「確認繼續」後，`_step_compute_target()` 執行。

| # | 物件 | 方法 | 輸入 | 輸出 |
|---|------|------|------|------|
| 8 | **TargetZCompute** | `compute(selected)` | pos_base_mm + yaw_deg | 接近目標（x, y, z+300mm, yaw） |

**注意：** 頭部相機偵測結果已在 base frame，不需要 Cam2Flange 補償，直接 z+300mm。

---

## 階段四：移動執行

觸發方式：使用者按「確認繼續」後，`_step_start_moving()` 執行。

### 分叉點

```
_step_start_moving()
        ↓
  _use_move_linear?
    ├─ False（預設）→ jog P-control 模式
    └─ True        → MoveLinear 服務模式（需要 AUTONOMOUS 模式）
```

---

### 模式 A：jog P-control（預設，MANUAL/AUTONOMOUS 皆可）

| # | 物件 | 方法 | 功能 |
|---|------|------|------|
| 9a | **TrajectoryPlan** | `plan(start, target)` | 規劃 Clamped Cubic B-spline 4D 軌跡（X,Y,Z,Yaw 同步插值） |
| 10a | **ExecuteMotion** | `compute(t, pos, rot, traj)` | 500Hz P-control，計算每幀 vel/rot 速度命令 |
| 11a | **ros2_node** | `publish_jog(vel, rot)` | 發布 `/kr/motion/jog_linear`（rclpy 直接發布，<1ms 延遲） |
| 12a | **CheckArrive** | `update(pos, rot, now)` | 確認手臂物理靜止 ≥ 0.3s 才判定到位 |

**P-control 參數：**

| 參數 | 值 |
|------|-----|
| 速度上限 | 30 mm/s（線速）、15 deg/s（轉速） |
| P gain XYZ | 5.0 |
| P gain Yaw | 5.0 |
| 位置容差 | 1 mm |
| 角度容差 | 1 deg |
| 穩定幀數 | 連續 10 幀在容差內 |
| 最短執行時間 | 軌跡總時間 × 80% |

---

### 模式 B：MoveLinear 服務（需要 AUTONOMOUS 模式）

| # | 物件 | 方法 | 功能 |
|---|------|------|------|
| 9b | **ros2_node** | `call_move_linear(pos, rot, speed)` | 呼叫 `/kr/motion/move_linear` 服務，Kassow 內部規劃軌跡，阻塞至 success=True |

**服務參數：**

| 參數 | 值 |
|------|-----|
| ref | 1（BASE frame） |
| ttype | 0（TT_VEL，速度追蹤） |
| tvalue | 速度滑桿值（mm/s） |
| chaining | 0（CH_INT，打斷前一個動作） |
| timeout | 60s |

## 階段五：手腕相機細調（⚠ 尚未接入，待實作）

到達器械上方後，切換至 D405 手腕相機做精細定位。

| # | 物件 | 方法 | 輸入 | 輸出 |
|---|------|------|------|------|
| 13 | **YoloDetectHandcam** | `infer(frame)` | D405 彩色幀 | bbox、center、conf、angle_deg |
| 14 | **GetDepthHandcam** | `get_depth(center, depth_img)` | 像素座標 + 深度圖 | `depth_mm`（5×5 patch 中位數） |
| 15 | **Pixel2HandCam** | `project_all(dets)` | (u,v) + depth_mm + D405 內參 | `pos_cam_mm`（手腕相機座標系） |
| 16 | **HandCam2Flange** | `transform_all(dets)` | pos_cam_mm + T_cam2gripper | `pos_flange_mm`（法蘭座標系） |
| 17 | **Angle2RzHandcam** | `convert_all(dets)` | angle_deg + T_cam2gripper | `yaw_deg`、`Rz`（法蘭座標系） |
| 18 | **Flange2Base** | `transform_all(dets)` | pos_flange_mm + 即時 TCP 位姿 | `pos_base_mm`（base frame） |
| 19 | **WhichFirstHandcam** | `get_first(dets)` | N 個器械（含 depth_mm） | 1 個最優先器械 |
| 20 | **TargetConsiderGripper** | `compute(selected)` | pos_base_mm + yaw_deg | 法蘭目標位姿（z + gripper_length） |

**手腕相機校正矩陣：** `T_cam2gripper_20260526_more14z.npy`（手腕相機→法蘭，4×4）

**WhichFirstHandcam 優先規則：**
- 主要：`depth_mm` 越小（越淺）越優先
- 次要：depth 差距 ≤ 15mm 時，`center[0]`（u，水平位置）越小越優先

## 階段六：夾取

| # | 物件 | 方法 | 功能 |
|---|------|------|------|
| 21 | **GripperControl** | `close()` | 呼叫 `/kr/iob/set_discrete_output`（DO01, index=1, value=1），等待 1.5s |
| 22 | **MemoryInstrumentPoint** | `record(target)` | 記錄器械位姿到槽位（最多 3 個），累計 `total_recorded` |

---

## 階段七：放置序列（三段移動）

| # | 物件 | 方法 | 輸入 | 輸出 |
|---|------|------|------|------|
| 23 | **PutSiteGet** | `get(total_recorded)` | 計數器（1-based） | 放置槽位位姿 `{x,y,z,yaw}` |
| 24 | **PlaceSequenceTargets** | `compute(pos, rot, put_site)` | 當前 TCP + 放置位姿 | `{lift, approach, place}` 三段目標 |
| 25 | **TrajectoryPlan** | `plan()` ×3 | 各段 start/target | B-spline 軌跡（重用） |
| 26 | **ExecuteMotion** | `compute()` | 500Hz | JogLinear 命令（重用） |
| 27 | **CheckArrive** | `update()` | 手臂位姿 | 到位確認（重用） |
| 28 | **GripperControl** | `open()` | — | 夾爪釋放（DO01, value=0），等待 1.5s |

**放置三段說明：**
1. `lift`：原地 Z +200mm（抬升，避免碰到旁邊器械）
2. `approach`：XY + Rz 橫移到放置點上方（Z 不動）
3. `place`：Z 下降到放置位置

**放置槽位（base frame）：**

| 計數 | x (mm) | y (mm) | z (mm) |
|------|--------|--------|--------|
| 第 1 個 | 513 | -150 | -355 |
| 第 2 個 | 513 | -110 | -355 |
| 第 3 個 | 513 | -70 | -355 |

---

## 階段八：回 Home

| # | 物件 | 方法 | 輸入 | 輸出 |
|---|------|------|------|------|
| 29 | **ReturnHomeTargets** | `compute(pos, rot)` | 當前 TCP | `{lift, home}` 兩段目標 |
| 30 | **TrajectoryPlan / ExecuteMotion / CheckArrive** | 同上 | — | 依序執行（重用） |

**Home 位姿：** (448.5, -245.4, 69.7) mm，yaw = 0°

## 完整資料流圖

```
━━━━━━━━━━━━━━━━━━ 【頭部相機偵測】━━━━━━━━━━━━━━━━━━
YoloEngine (D435I, GPU)  ←── RoiFilter（ROI 範圍過濾）
    │ bbox, center, conf, angle_deg
DepthReader  →  Pixel2HeadCam  →  HeadCam2Base  →  Angle2Rz
    │ pos_base_mm, yaw_deg（base frame）
    ▼
yaw offset（-45°）+ 180° 對稱選優解
    ▼
WhichFirst（Z 越大越優先，Z 相同才看 Y）
    ▼
TargetZCompute（z + 300mm）
    │ 接近目標
    ▼
[移動] TrajectoryPlan → ExecuteMotion(500Hz) → CheckArrive
       或 call_move_linear（MoveLinear 服務）

━━━━━━━━━━━━━━━━━━ 【手腕相機細調 ⚠未接入】━━━━━━━━━━━━━━
YoloDetectHandcam (D405)
    │
GetDepthHandcam → Pixel2HandCam → HandCam2Flange
    → Angle2RzHandcam → Flange2Base
    │ pos_base_mm（base frame，透過即時 TCP 轉換）
    ▼
WhichFirstHandcam（depth 淺優先）
    ▼
TargetConsiderGripper（z + gripper_length）
    ▼
[移動] TrajectoryPlan → ExecuteMotion → CheckArrive

━━━━━━━━━━━━━━━━━━ 【夾取】━━━━━━━━━━━━━━━━━━━━━━━━━━
GripperControl.close()  +  MemoryInstrumentPoint.record()

━━━━━━━━━━━━━━━━━━ 【放置】━━━━━━━━━━━━━━━━━━━━━━━━━━
PutSiteGet → PlaceSequenceTargets（lift → approach → place）
    ↓ ×3 段
TrajectoryPlan → ExecuteMotion → CheckArrive
    ↓
GripperControl.open()

━━━━━━━━━━━━━━━━━━ 【回 Home】━━━━━━━━━━━━━━━━━━━━━━━
ReturnHomeTargets（lift → home）
    ↓ ×2 段
TrajectoryPlan → ExecuteMotion → CheckArrive
    ↓
回到頭部相機偵測（下一輪）
```

---

## 執行執行緒架構

```
Main Thread（DearPyGui 渲染迴圈）
  ├─ tick() 每幀呼叫
  │    └─ _try_inject_detection()（限速 0.3s）
  │         └─ 座標轉換鏈 + WhichFirst + phase 切換
  │
  └─ _on_confirm()（按鈕回調）
       └─ _step_start_moving()
            ├─ jog 模式：daemon thread（_motion_loop，500Hz）
            │    ├─ TrajectoryPlan.sample(t)
            │    ├─ ExecuteMotion.compute()
            │    ├─ publish_jog()
            │    └─ CheckArrive.update()
            │
            └─ MoveLinear 模式：background thread（_run_move_linear）
                 └─ call_move_linear()（阻塞至 success）
```

## 各物件簡述（完整）

| 物件 | 職責（一句話） | 狀態 |
|------|--------------|------|
| YoloEngine | 雙相機 GPU 推論引擎，回傳偵測框和傾角 | ✅ |
| RoiFilter | ROI 範圍過濾，判斷偵測中心是否在設定範圍內 | ✅ |
| DepthReader | 從頭部相機深度圖讀取指定像素深度值（5×5 中位數） | ✅ |
| Pixel2HeadCam | 像素座標 + 深度 → 頭部相機座標系 3D 點 | ✅ |
| HeadCam2Base | 頭部相機座標系 → base frame（T_matrix 手眼校正） | ✅ |
| Angle2Rz | 器械 2D 傾角 → base frame 偏航角 Rz | ✅ |
| WhichFirst | 選最優先器械（Z 越大越優先，Z 相同才以 Y 決定） | ✅ |
| TargetZCompute | 計算器械上方 300mm 的接近點 | ✅ |
| TrajectoryPlan | 規劃 Clamped Cubic B-spline 4D 軌跡（X,Y,Z,Yaw 同步） | ✅ |
| ExecuteMotion | 500Hz P-control 計算每幀 JogLinear 速度命令 | ✅ |
| CheckArrive | 確認手臂物理靜止 ≥ 0.3s（到位雙重確認） | ✅ |
| ros2_node | rclpy 直接發布 JogLinear / 呼叫 MoveLinear 服務 | ✅ |
| YoloDetectHandcam | 手腕相機（D405）獨立 YOLO 推論引擎 | ⚠ 未接入 |
| GpuInferenceSwitch | GPU 推論使用權切換（頭部/手腕相機互斥） | ⚠ 未接入 |
| GetDepthHandcam | 從手腕相機深度圖讀取深度值（5×5 中位數） | ⚠ 未接入 |
| Pixel2HandCam | 像素座標 + 深度 → 手腕相機座標系 3D 點（D405 內參） | ⚠ 未接入 |
| HandCam2Flange | 手腕相機座標系 → 法蘭座標系（T_cam2gripper） | ⚠ 未接入 |
| Angle2RzHandcam | 手腕相機 2D 傾角 → 法蘭座標系 Rz（T_cam2gripper） | ⚠ 未接入 |
| Flange2Base | 法蘭座標系 → base frame（即時 TCP 位姿，每幀需更新） | ⚠ 未接入 |
| WhichFirstHandcam | 手腕相機優先順序（depth 淺優先，相近時靠左優先） | ⚠ 未接入 |
| TargetConsiderGripper | 考慮夾爪長度（z + 120mm），計算法蘭真正目標位姿 | ✅ |
| GripperControl | 夾爪開合（DO01）：close()/open()，透過 ROS2 service | ✅ |
| MemoryInstrumentPoint | 記錄已夾取器械位姿（最多 3 個槽位） | ✅ |
| PutSiteGet | 根據計數器決定放置槽位位姿（3 個固定點） | ✅ |
| PlaceSequenceTargets | 計算放置三段目標（抬升→橫移→下降） | ✅ |
| ReturnHomeTargets | 計算回 Home 兩段目標（抬升→Home） | ✅ |
