## 一、MoveLinear 移動實作（取代 jog P-control）

### 問題排查過程

**階段 1：模式問題**
- 發現 Kassow MoveLinear 需要切換到 AUTONOMOUS 模式才能執行
- 新增 `/kr/motion/set_interactivity` 服務呼叫
  - `enable=False` → AUTONOMOUS（可執行 MoveLinear）
  - `enable=True` → MANUAL（可執行 Jog）
- 每次 MoveLinear 前後自動切換模式，`finally` 區塊確保一定恢復 MANUAL

**階段 2：座標系問題（關鍵發現）**
- 初始使用 `ref=1 (BASE)`，導致手臂暴衝旋轉
- 排查後確認：`SystemState.pos` 和 `get_robot_pose` 回傳的是 **WORLD frame**，非 BASE frame
- WORLD 和 BASE 之間有 X 軸 90° 旋轉關係（實測：同一 TCP，WORLD Y ≈ -BASE Z，WORLD Z ≈ BASE Y）
- 修正為 `ref=0 (WORLD)` 後，10mm 移動測試成功

**階段 3：姿態參數問題**
- 初始 `rot=[0.0, 0.0, 0.0]` 導致 IK 失敗（目標姿態不可達）
- 修正：帶入手臂當前 `current_rot`，保持 R/P，只修改 Yaw

**階段 4：threading 問題**
- 原本使用 `rclpy.spin_until_future_complete`，與 rclpy_spin 執行緒衝突
- 改用 `threading.Event` + `future.add_done_callback`，由 spin 執行緒處理 callback

### 最終結果
- MoveLinear 正常運作，設為預設移動方式
- Jog P-control 改為勾選「改用 Jog P-control」才觸發
- 預設速度：130 mm/s（GUI 滑桿可調）

## 二、HandcamAngle2Yaw 符號修正

### 問題
- 偵測器械傾斜時，計算出的 Rz 與實際需要的 Rz 有系統性誤差
- 實測：fitEllipse=68.8° 計算出 -66.2°，但實際需要 -21°

### 實測 4 個點確認規律

| fitEllipse (°) | 計算結果（舊） | 正確 Rz | 規律 |
|---|---|---|---|
| 5.8 | -129.2° | -140.8° | -135 - 5.8 |
| 37.4 | -97.6° | -172.4° | -135 - 37.4 |
| 85.1 | -49.9° | 139.9° | normalize(-135 - 85.1) |
| 134.5 | -0.5° | -89.5° | normalize(-135 - (134.5-180)) |

### 修正
- 公式從 `+angle_deg + offset` 改為 **`-angle_deg + offset`**（符號相反）
- fitEllipse 座標與 Rz 旋轉方向相反，加負號後所有角度均正確

## 四、放置序列 Callback Bug 修正

### 問題
- `_step_start_sequence` 把最終 callback（如 `_step_after_place`）存入 `self._move_on_arrived`
- `_step_start_moving` 馬上將 `self._move_on_arrived` 覆蓋為 `_step_next_in_sequence`
- 導致序列最後一段完成後，最終 callback 永遠不被呼叫

### 修正
- 新增 `self._sequence_done_cb` 欄位獨立儲存序列完成 callback
- `_step_next_in_sequence` 隊列清空時改從 `_sequence_done_cb` 呼叫

## 五、BoxSmoother 滑行幀（coast_frames）

### 問題
- 頭部相機 YOLO 偵測在某些幀完全沒有輸出（非低信心分，而是直接沒輸出）
- 只要一幀沒偵測，`_BoxSmoother` 立刻清空，bbox 閃爍消失

### 修正
- 新增 `coast_frames=3` 參數
- YOLO 沒輸出時保留上一幀 bbox 顯示（標記 stable=False），連續 3 幀都沒偵測才真正清空

## 已知待改善缺點

1. **手腕相機 Z 值不穩定**：法蘭面伸下去的 z 時常過高或過低，誤差無規律（深度讀取不穩定）
2. **需要改成全自動化**：移除手動確認步驟
3. **前後半段目標不一致**：頭部相機瞄準 A，手腕相機卻夾到 B（WhichFirst 邏輯不同步）
4. **D405 USB 斷線**：手臂移動時 USB 纜線被拉扯，Pipeline 停止
5. **手腕相機偵測較慢**：BoxSmoother confirm_frames=3 需等 300ms
