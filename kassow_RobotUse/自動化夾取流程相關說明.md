# Kassow KR810 自動夾取系統 — 完整開發進度與流程說明

更新：2026-06-15

---

## 一、系統概述

Kassow KR810 右臂 + 雙 RealSense 相機，執行手術器械的全自動辨識、夾取、放置與還原。

**完整一輪流程：**
```
頭部相機偵測 → 接近器械上方 → 手腕相機細調 → 夾取
→ 放置到托盤 → 回 Home（×3 支器械）
→ 逆序還原 3 支器械 → 回 Home → 自動開始下一輪（無限循環）
```

## 四、完整自動夾取流程

### 狀態機與等待時間

```
【前半段：頭部相機偵測 → 接近】

detecting
  ↓ YOLO (D435I, GPU) 持續偵測，BoxSmoother 穩定 3 幀
confirm_selection ── 固定等 3s（bbox 穩定）
  ↓ 座標轉換鏈（見第五節）→ 接近目標（z+300mm）
confirm_target ──── 等 0.3s
  ↓ MoveLinear 移動到器械正上方 300mm
moving_approach → 到位
confirm_arrived ─── 等 0.3s

【後半段：手腕相機細調 → 夾取】

handcam_detecting ── 固定等 5s 收集偵測結果
  （5s 內無結果 → stopped）
confirm_handcam ─── 等 0.3s
  ↓ 座標轉換鏈（見第六節）→ 夾取位姿
confirm_grasp_target 等 0.3s
  ↓ MoveLinear 移動到夾取位置
moving_grasp → 到位
confirm_grasp_arrived 等 0.3s
  ↓ 夾爪閉合
closing_gripper ─── 固定等 1.5s

【放置】

confirm_gripper_closed 等 0.3s
  ↓ 記錄器械位置（MemoryInstrumentPoint） + 計算放置點
confirm_recording ── 等 0.3s
  ↓ 三段移動：① Z 抬升 200mm → ② XY 橫移 → ③ Z 下降放置
moving_sequence（×3 段）→ 到位
confirm_place_arrived 等 0.3s
  ↓ 夾爪張開，等 1.5s
  ↓ 回 Home（Z 抬升 → Home: 685.1, -245.5, -75mm）

（重複以上 × 3 支器械）

【還原序列】

Home 等待 2s
  ↓ RestoreInstrumentSequence（逆序 3→2→1）
  ↓ 每支器械 8 步：接近放置台 → 夾取 → 移到托盤原位上方 → 放回原位 → 張夾 → 抬升
  （共 8 步 × 3 支，三支全部完成後才回一次 Home）

restore_complete 等待 2s
  ↓ 自動呼叫 _on_start() → 重新開始下一輪
```

## 五、座標轉換鏈 — 頭部相機（前半段）

觸發：`_try_inject_detection()`，每 0.3s 執行一次。

| # | 物件 | 輸入 → 輸出 |
|---|------|------------|
| 1 | **YoloEngine** (cam0) | D435I 彩色幀 GPU 推論 → bbox、center、conf、angle_deg |
| 2 | **DepthReader** | 像素座標 + 深度圖 → depth_mm（5×5 patch 中位數） |
| 3 | **Pixel2HeadCam** | (u,v) + depth_mm + D435I 內參 → pos_cam_mm |
| 4 | **HeadCam2Base** | pos_cam_mm + T_matrix → pos_base_mm（WORLD frame） |
| 5 | **Angle2Rz** | angle_deg + T_matrix → yaw_deg |
| 6 | yaw offset 邏輯 | yaw_deg + 當前 TCP yaw → 調整後 yaw（offset=-45°，180° 選優解） |
| 7 | **WhichFirst** | N 個器械 → 最優先 1 個（Z 越大越優先，Z 相同以 Y 決定） |
| 8 | **TargetZCompute** | pos_base_mm → 接近目標（z + 300mm） |
| 9 | **PoseOffset** | 接近目標 → x + 25mm 補償 |
| 10 | **Cam2Flange** | 相機目標位置 → 法蘭目標位置（補償手腕相機與法蘭的安裝偏移，使用 T_cam2gripper） |

## 六、座標轉換鏈 — 手腕相機（後半段）

觸發：到達器械上方後，handcam_detecting 階段持續 5s。

| # | 物件 | 輸入 → 輸出 |
|---|------|------------|
| 1 | **YoloEngine** (cam1) | D405 彩色幀 GPU 推論 → bbox、center、conf、angle_deg |
| 2 | **GetDepthHandcam** | 像素座標 + 深度圖 → depth_mm（7 幀跨幀中位數，見附註） |
| 3 | **Pixel2HandCam** | (u,v) + depth_mm + D405 內參 → pos_cam_mm |
| 4 | **HandCam2Flange** | pos_cam_mm + T_cam2gripper → pos_flange_mm |
| 5 | **Angle2RzHandcam** | angle_deg + T_cam2gripper → yaw_deg（法蘭座標系） |
| 6 | **Flange2Base** | pos_flange_mm + 即時 TCP → pos_base_mm（WORLD frame） |
| 7 | **WhichFirstHandcam** | N 個器械 → 最優先 1 個（depth_mm 越小越優先；差距 ≤15mm 時以畫面 u 座標靠左優先） |
| 8 | **TargetConsiderGripper** | pos_base_mm → z + gripper_length（GUI 可調，目前設定 100mm） |
| 9 | **GraspZOverride** | 覆蓋 z → **-394.5mm**（寫死，繞過深度不穩定） |
| 10 | **HandcamAngle2Yaw** | angle_deg → yaw_base_deg（公式：`-angle_deg + (-135°)`） |

**附註 — GetDepthHandcam 時間濾波（2026-06-15 加入）：**
`buffer_size=7`，保留最近 7 幀的深度讀取值，回傳「7 幀中位數」，抑制 D405 因器械反光導致的跳動。每輪夾取開始前自動 `reset_buffer()`。

## 七、移動執行方式

**預設：MoveLinear 服務模式**

```
set_interactivity(False)  → 切換到 AUTONOMOUS 模式
call_move_linear(pos, rot, speed=200mm/s, ref=0(WORLD))
set_interactivity(True)   → 恢復 MANUAL 模式（finally 區塊保證執行）
```

**重要：**
- `ref=0` 是 WORLD frame，`ref=1 (BASE)` 會導致手臂暴衝旋轉
- `rot` 必須帶入當前手臂 `current_rot`，不能用 `[0,0,0]`（否則 IK 失敗）
- threading 使用 `threading.Event` + `future.add_done_callback`（非 spin_until_future_complete）

備用：勾選「改用 Jog P-control」可切回 500Hz P-control + B-spline 軌跡模式。

## 九、器械還原（RestoreInstrumentSequence）

三支器械全數放置完成後，依**逆序（3→2→1）**將每支器械夾回原先在托盤的位姿。

**記錄時機：** 夾爪閉合 1.5s 後（`_step_record_and_compute_place`）
**記錄來源：** `_handcam_selected`（手腕相機偵測到的實際位置）
**記錄內容：**
- `pos_base_mm`：WORLD frame 的 [x, y, z]
- `yaw_deg`：HandcamAngle2Yaw 計算出的法蘭 Rz

**每支器械 8 步（逆序 3→2→1）：**

① 移到放置台上方（z+300mm，yaw=-135°）
② 移到放置台夾取高度（z=-397mm，yaw=-135°）
③ 夾爪閉合
④ 抬升離開放置台（z+300mm）
⑤ 移到托盤原始位置上方（z+300mm，原始 yaw）
⑥ 移到托盤原始夾取高度（z=-394.5mm，原始 yaw）
⑦ 夾爪張開
⑧ 抬升離開托盤（z+300mm）

三支全數完成後 → 回 Home（685.1, -245.5, -75mm）（僅一次，不是每支器械都回）

還原完成後 `MemoryInstrumentPoint.clear() + reset_counter()`，下一輪重新記錄。

## 十一、已知待處理問題

### 高優先
- **手腕相機深度不穩定**：D405 近距離（~200mm）拍攝時器械反光面深度大量為 0。已加 7 幀時間濾波；目前以 `GraspZOverride` 寫死 Z=-394.5mm 繞過，非根本解。方向：降低 D405 IR 雷射功率、或改用運動學深度估算。
- **手腕相機夾取順序不穩定**：`WhichFirstHandcam` 用 depth_mm 排序，但深度讀取不穩導致 depth_mm 每次不同，排序結果無法預期。需改為鎖定畫面中心最近的 bbox。

### 中優先
- **前後半段器械不一致**：頭部相機偵測到 A，手腕相機卻夾到 B。需讓後半段鎖定前半段已定位的同一支器械。
- **D405 USB 斷線**：手臂移動時 USB 纜線被拉扯，pipeline 停止，需改善走線或加重連邏輯。

### 低優先
- **手腕相機偵測較慢**：BoxSmoother `confirm_frames=3` 需等 300ms，可降為 1~2。
- **螢幕每 1~3 小時凍結**：疑似 RTX 5060 GPU driver crash/recovery。

## 十三、Kassow ROS2 重要說明

- **座標系**：`SystemState.pos` 和 `get_robot_pose` 回傳 **WORLD frame**，非 BASE frame。MoveLinear 必須用 `ref=0 (WORLD)`。
- **模式切換**：`/kr/motion/set_interactivity` — `enable=False` → AUTONOMOUS（才能執行 MoveLinear）；`enable=True` → MANUAL（才能執行 Jog）。
- **ROS2 環境**：需先 source `/opt/ros/jazzy/setup.bash` 和 `~/ros2_ws/install/setup.bash`（`run.sh` 已處理）。
