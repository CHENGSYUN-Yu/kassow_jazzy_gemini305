## 二、D405 深度穩定性改善（時間濾波器 + 洞填充濾波器）

### 問題
- 手腕相機偵測的法蘭面目標 Z 值不穩定，每次誤差無規律（推測是近距離拍攝反光導致）

### 嘗試
- 在 `realsense.py` `_Camera._capture_loop()` 加入 librealsense 內建濾波器串聯：
  ```
  raw depth_frame
      ↓ rs.temporal_filter()   （跨幀移動平均）
      ↓ rs.hole_filling_filter()（填補 0 值區域）
  ```
- 兩個濾波器在 `connect()` 初始化，對兩台相機都套用

### 結果
- 深度穩定性有所改善，但仍有不規律誤差（見待改善問題）
- **暫時解法**：新增 `GraspZOverride` 物件，將夾取目標 Z 寫死為 **-398.2mm**，完全繞過深度偵測結果（見第三節）

## 四、器械還原功能（RestoreInstrumentSequence 物件）

### 功能
三支器械全數放置完成後，依逆序（3→2→1）將每支器械夾回原先在托盤的位姿。

### 新增物件
- `src/restore_instrument_sequence.py`：計算還原動作序列（8 步 × 3 支 + 最後回 Home）
  - 動作類型：`move`、`close_gripper`、`open_gripper`
  - 放回托盤時使用原始偵測的 `yaw_deg`（HandcamAngle2Yaw 結果）
  - Rx/Ry 沿用手臂當前姿態（不記錄，不需要）

### 執行機制
- `AutoGrasp._start_restore_sequence()`：計算序列並啟動
- `AutoGrasp._execute_next_restore_action()`：逐步執行動作佇列
- `_step_record_and_compute_place` 改為記錄 `_handcam_selected`（實際器械位置），不再記錄 `_auto_selected`（可能是 Home 位置）

### 位姿記憶（`MemoryInstrumentPoint`）

**記錄時機**：`_step_record_and_compute_place()`，即夾爪閉合 1.5s 後

**記錄來源**：優先用 `_handcam_selected`（手腕相機偵測到的器械位置），無則退回 `_auto_selected`

**記錄內容**（每支器械，存入 slot 0~2）：
- `pos_base_mm`：器械在 WORLD frame 的 [x, y, z]（手腕相機座標轉換鏈輸出）
- `yaw_deg`：HandcamAngle2Yaw 計算出的法蘭 Rz（夾取當下的旋轉）

---

### 位姿讀取（還原序列用）

**讀取時機**：`_start_restore_sequence()`，即三支器械全數放置完、Home 等待 2s 後

**讀取方式**：
```python
original_records = [self._mem.get(i) for i in range(3)]  # slot 0,1,2
put_sites        = [self._put_site.get(i+1) for i in range(3)]  # 放置點 1,2,3
```

**還原目標計算（RestoreInstrumentSequence.compute）**：
- 從 `original_records[i]['pos_base_mm']` 取得托盤原始 xyz
- 從 `original_records[i]['yaw_deg']` 取得原始 Rz
- 接近高度：原始 z + 300mm
- 放回高度：`return_z_mm`（= GraspZOverride.z_mm = -398.2mm）
- 夾取放置台高度：`put_site['z_mm']`（放置點 z = -397mm）

**記憶體重置**：還原完成後 `clear()` + `reset_counter()`，下一輪重新記錄

## 重要待改善問題

### 1. 手部相機深度偵測不穩定
- **現象**：每次伸下去夾取的法蘭面 Z 值不穩定，過高或過低，誤差無規律
- **推測原因**：D405 近距離拍攝（~200mm）時器械表面反光，結構光深度偵測失靈，深度圖出現大量 0 值或噪點
- **已嘗試**：temporal_filter + hole_filling_filter（改善有限）
- **目前處理**：`GraspZOverride` 暫時寫死目標 Z，繞過深度不穩定問題
- **根本修法方向**：降低 D405 IR 雷射功率、或改用其他深度估算方式（運動學計算）

### 2. 手部相機優先夾取算法不穩定
- **現象**：每次偵測到多支器械時，優先夾取的順序無法被預料，不符合「最靠近相機」的規律
- **原因**：`WhichFirstHandcam` 的排序邏輯（目前用 Z 最大優先）與手腕相機視角下的「最近」定義不一致；加上深度讀取不穩定，Z 值每次不同，導致排序結果不穩定
- **待改善**：需重新設計手腕相機下的優先判斷邏輯，或直接鎖定最靠近畫面中心的 bbox
